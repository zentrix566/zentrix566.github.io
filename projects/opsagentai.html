OpsAgentAI

基于 RAG 架构与 Dify 编排的 DevOps 故障智能自愈平台

技术文章 GitHub 源码

项目背景

针对传统运维告警"仅通知、不诊断"的痛点，解决 CI/CD 流水线报错日志冗长、依赖人工排查导致的 MTTR（平均修复时间）过长问题。 通过整合内部运维 SOP 手册与 LLM 能力，构建一套从告警捕获到修复建议推送的自动化闭环系统。

核心职责

  • 架构设计与实现

    主导基于 Dify 的 AI Agent 流程编排，利用 Python 开发中转网关，对接 GitHub Actions/K8s 告警源，实现告警信息的自动化清洗与上下文提取。

  • 知识库（RAG）优化

    针对非结构化运维文档进行精细化语义切分，采用 混合检索（Hybrid Search） 方案，将 LLM 在专业运维领域的回答准确率提升至 90% 以上。

  • 交互闭环集成

    深度集成 飞书卡片消息（Action Card），设计"现象解析-根因分析-修复建议-人工确认"的一键式交互链路，支持移动端快速复核排查结论。

  • 工程化保障

    设计日志预处理逻辑，解决大宗告警日志超过 Token 限制的问题，并引入人工审批环节规避 AI 误操作风险。
